# 📘 AI Engineering: RAG + Agentic Systems

## 🎯 Contexto

Com a evolução dos LLMs, empresas passaram a integrar modelos de linguagem com dados externos e agentes autônomos, criando sistemas mais inteligentes, atualizados e confiáveis.

Duas abordagens dominam esse cenário:

- RAG (Retrieval-Augmented Generation)
- Agentic Systems (IA com tomada de decisão e múltiplos passos)

Este projeto utiliza o NotebookLM como ferramenta de aprendizagem ativa para compreender, estruturar e aplicar essas arquiteturas modernas de IA.

## 🚀 Objetivos

- Dominar o conceito de RAG e Agentic AI
- Entender pipelines reais de sistemas inteligentes
- Desenvolver engenharia de prompts avançada
- Criar material reutilizável para estudos
- Demonstrar capacidade técnica alinhada ao mercado de IA

## 📌 RAG (Retrieval-Augmented Generation)

### 🔹 O que é?
Integra busca de informação com geração de IA.

### 🔹 Pipeline

1. Input do usuário
2. Busca em base externa
3. Recuperação de dados relevantes
4. Geração com LLM

---

## 📌 Agentic Systems

### 🔹 O que é?
Sistemas de IA capazes de tomar decisões em múltiplos passos.

### 🔹 Características

- Planejamento
- Uso de ferramentas
- Execução iterativa

---

## 🔥 Diferença chave

- RAG → responde melhor
- Agent → resolve problemas
